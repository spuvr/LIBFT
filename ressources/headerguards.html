Header Guards in C/C++

What Are Header Guards?

Header guards are preprocessor directives used in header files to ensure that the content of the header is included only once during compilation, even if the header is included multiple times in different files. 

#ifndef HEADER_NAME_H
#define HEADER_NAME_H

// Declarations and definitions

#endif // HEADER_NAME_H

Why Do We Need Header Guards?

  • Avoid Multiple Inclusions: Prevents the same file from being included multiple times.
  • Prevent Redefinition Errors: Stops errors caused by multiple definitions of structs, functions, or variables.

Are They Needed Only for struct?

No, header guards are needed for everything in a header file, such as:

  • Structs
  • Function Prototypes
  • Macros
  • Global Variables
  • Typedefs
  • Enums

Example Without Header Guards

struct MyStruct {
    int x;
    int y;
};

If the file containing this struct is included multiple times, it will throw a redefinition error.

Solution

Use header guards:

#ifndef HEADER_H
#define HEADER_H

struct MyStruct {
    int x;
    int y;
};

#endif // HEADER_H
